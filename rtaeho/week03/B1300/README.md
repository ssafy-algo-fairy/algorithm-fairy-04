### 📌 문제 정보

- **번호:** 1300
- **제목:** K번째 수
- **난이도:** Gold 1
- **분류:** 이분 탐색

---

### 📝 문제 요약

N×N 배열 B에서 `B[i][j] = i * j`로 정의할 때, 이를 1차원으로 정렬한 배열 A의 K번째 값을 구하는 문제  
배열 전체를 직접 생성하면 N이 최대 10^5이라 메모리 초과 → 이분탐색으로 접근

---

### 💡 접근 방식

> "mid 이하인 값이 배열에 몇 개 있는가?"를 O(N)으로 계산할 수 있다면, 이분탐색으로 K번째 값을 찾을 수 있음  
> 배열을 실제로 만들지 않고, 조건 함수만으로 정답을 좁혀나가는 Parametric Search 방식

---

### 🔹 1단계 – i행에서 mid 이하인 값 개수 계산

i행은 `i, 2i, 3i, ..., Ni` (i의 배수들)로 구성  
이 중 `mid` 이하인 값의 개수 = `j × i ≤ mid`를 만족하는 j의 수 = `mid / i` (정수 나눗셈)

단, 행에는 값이 N개만 존재하므로 N을 초과할 수 없음

```
N=3, mid=7 기준:
1행: min(7/1, 3) = 3
2행: min(7/2, 3) = 3
3행: min(7/3, 3) = 2
합계 count = 8
```

```java
for (int i = 1; i <= N; i++) {
    count += Math.min(mid / i, N);
}
```

---

### 🔹 2단계 – 이분탐색으로 최솟값 탐색

`count >= k`를 만족하는 **가장 작은 mid** 를 찾는 것이 목표

- `count < k` → mid가 너무 작음 → `left = mid + 1`
- `count >= k` → 정답 후보 저장 후 더 작은 값 탐색 → `answer = mid`, `right = mid - 1`

```
N=3, k=7 탐색 예시:
mid=4 → count=6 < 7  → left=5
mid=7 → count=8 ≥ 7  → answer=7, right=6
mid=5 → count=7 ≥ 7  → answer=5, right=4
종료 → 최종 answer=5
```

탐색 범위: `left=1`, `right=k` (K번째 수는 최대 k)

---

### 💻 핵심 코드

```java
while (left <= right) {
    long mid = (left + right) / 2;
    long count = 0;
    for (int i = 1; i <= N; i++) {
        count += Math.min(mid / i, N);
    }

    if (count < k) {
        left = mid + 1;
    } else {
        answer = mid;
        right = mid - 1;
    }
}
```

---

### ⏳ 복잡도 분석

- **시간 복잡도:** `O(N log K)`
  - 이분탐색 O(log K) × 각 단계에서 count 계산 O(N)
- **공간 복잡도:** `O(1)`
  - 배열 없이 변수만 사용

---

### ⚠️ 어려웠던 점

처음에 이분탐색을 배열에서 값을 찾는 용도로만 생각해서, "배열도 없는데 어떻게 이분탐색을 하지?"라고 막혔습니다.  
핵심은 배열을 만드는 게 아니라 "mid 이하가 몇 개인지" 세는 함수를 만드는 것이었습니다.  
`right = mid - 1`도 처음엔 1씩 줄이는 건지 헷갈렸는데, mid가 매번 절반으로 계산되니 실제로는 범위가 빠르게 좁혀지는 구조였습니다.  
또 `count >= k`일 때 바로 종료하지 않고 계속 탐색하는 이유가 "더 작은 정답이 있을 수 있기 때문"이라는 것, lower_bound 패턴임을 이해하고 나서야 전체 흐름이 잡혔습니다.
